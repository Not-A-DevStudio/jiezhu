# 不仅没有跑路，反而还在长期维护！

为了长期维护 Jiezhu 项目的两个 GitHub 仓库——[宣传官网](https://github.com/not-a-devstudio/jiezhu/) 与 [核心库](https://github.com/not-a-devstudio/libjiezhu/)，我们决定发布这篇博客，让社区更清楚地了解我们未来的真实计划和目标。

## Jiezhu 与 Not A DevStudio 的起源

Jiezhu 项目灵感源自ChatGPT “雷霆万钧”的中文语法，完全不符合中国人的语言习惯。于是，我们 Not A DevStudio 的创始人决定把这种独特的语法风格带给每一个 AI。在确定 `Jiezhu v0.1.0` 的功能后，团队迅速组建，Jiezhu 项目正式启动——这也成为了 Not A DevStudio 的起点。

在开发早期，Jiezhu 原本计划在今年的愚人节上线，但我们实在等不及了，提前了几个月发布。所以，其实**Jiezhu 的气质本质上更接近“整蛊”**——我们把它包装成了一个正经项目，但它确实具备我们目前宣传的功能，且没有任何恶意。我们希望借此给用户带来一些惊喜和乐趣，同时也展示团队的创造力和幽默感。

正因如此，官网目前展示的 Roadmap 内容其实并不“实在”，甚至可以说是“整蛊式”的。里面提到的许多功能和计划，未必会真正实现。更准确的说法是：**只要社区愿意为某个功能提交 PR，我们就会认真考虑将其合并进来**。我们不会严格按照 Roadmap 的时间表推进开发，而是根据社区的参与和兴趣来动态调整优先级。我们希望通过这种方式，让 Jiezhu 成为一个真正由社区驱动的项目。

当然，Not A DevStudio 团队依然会承担大部分维护和开发工作。我们会优先选择贴近开发者需求的功能进行实现——有些内容虽然没有写在官网 Roadmap 上，但一直在持续迭代。我们希望通过这种灵活的方式，确保 Jiezhu 项目持续演进，同时也欢迎社区成员提出任何想法或建议，无论是功能还是项目管理方面，我们都会认真倾听并尽力落地。

以下 Roadmap 名称来自于 Github 仓库名，`libjiezhu` 是核心库，`jiezhu` 是宣传官网。我们会分别介绍它们的 Roadmap 内容。

## libjiezhu Roadmap v2026

[libjiezhu](https://github.com/not-a-devstudio/libjiezhu/) 的 Roadmap 主要分为两个阶段：2026 Q2 和 2026 Q4 及更远。

### 2026 Q3

1. **完善文档**  
   未来几个月内，我们将继续完善 Jiezhu 的文档体系，包括 API 文档、使用指南和开发者文档。
   此外，我们还计划部署一个全新的文档网站，结构化地集中展示所有文档内容。同时，代码中的注释也会大幅更新，提升可读性与可维护性。有关在线文档的具体内容请以上线时为准。

2. **接口兼容**  
   OpenAI 近期发布了全新的接口定义，我们会持续保持对 OpenAI 接口的兼容性。团队将密切关注 OpenAI 的更新，及时调整代码以适应变化，确保 Jiezhu 项目的稳定与可靠，让用户能够持续通过我们的项目调用 OpenAI API 兼容的服务。

### 2026 Q4 及更远

1. **功能扩展**  
   具体新功能正在开发中，敬请期待后续更新！我们计划提供更丰富的 API 接口、增加实用工具等。通过持续扩展功能，满足用户需求，提升 Jiezhu 的竞争力和吸引力。  
   当然，如果你有好的功能需求或建议，欢迎在 GitHub 上提交 issue 或 PR 告诉我们！

## jiezhu Roadmap v2026

[jiezhu](https://github.com/not-a-devstudio/jiezhu/) 的 Roadmap 是持续性的。

目前我们计划持续更新博客内容，分享项目的最新动态、技术细节和开发经验。我们会定期发布关于项目进展、功能更新和社区活动的博客文章，让大家第一时间了解 Jiezhu 的最新动态。

## BREAKING CHANGE

1. **Jiezhu 官网仓库即将修改LISCENCE**  
   目前 Jiezhu 官网仓库使用的是 MIT License，这意味着包括我们的博客在内的全部内容都是 MTI 许可，我们计划将**博客内容**迁移到 CC BY-NC-SA 4.0 License。这个变更主要是为了更好地保护我们的创意内容，同时允许社区在非商业用途下自由使用和分享我们的博客文章和其他创作内容。  
   需要注意的是，这个变更**只会影响博客内容**，Jiezhu 官网的**其他部分（如代码、文档等）仍然保持 MIT License 不变**。我们会在迁移过程中做好充分的沟通和说明，确保社区成员了解这个变更，并且能够继续享受我们提供的内容和资源。
